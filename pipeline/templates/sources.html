Methodology

This tracker follows two official voices for each country: its foreign ministry and its executive office — the Federal Chancellery in Germany, the Élysée (presidency) in France, and the Prime Minister's Chancellery in Poland. It ingests their press releases and official statements in German, French, and Polish, straight from each institution's native-language newsroom rather than the thinner English-translation sections. Each item is read by a language model, which classifies it for Weimar Triangle relevance and, for relevant items, extracts a one-sentence position summary in English. Following six institutions in three languages is exactly the work this site exists to save you.

Relevance classification

Because every source is a specific foreign ministry or executive office, the source country always counts as an actor — a Polish statement about Ukraine counts Poland even when the text never names it. On top of that, the same language model that reads each item identifies which other Weimar countries it concerns, which tracked issue areas it touches (Ukraine, defence, hybrid threats, enlargement, green transition, rule of law), and whether it explicitly invokes the Weimar Triangle or trilateral format.

An item is marked weimar_relevant when it is a foreign-ministry or executive-office item touching a tracked issue area, when two or more Weimar countries are involved on a tracked issue area, or when it carries an explicit trilateral reference. A stronger trilateral_signal flag is set when all three actors are present or the Weimar/trilateral reference appears explicitly. (Earlier editions matched keywords for this; that missed the German, French, and Polish wording and could not read paraphrase, so the reading is now done by the model.)

Positional comparison

The Recent page groups weimar-relevant events into clusters where two or more countries published on the same topic within a 14-day window. This surfaces cases where Germany and Poland both speak about Ukraine in the same week — even without a joint statement — enabling comparison of whether their positions converge or diverge.

Stance scoring

Every statement is rated against the officially agreed Weimar Triangle goals (from the joint declarations of February and May 2024) on a five-point scale, the same way political scientists hand-code party manifestos:

  • +2 — actively advances the goal: concrete commitments, resources, initiatives
  • +1 — supports the goal rhetorically, without new commitments
  • 0 — neutral: mentions the topic without taking a stance
  • −1 — hedges: partial support with significant caveats
  • −2 — opposes or undermines the goal

Each rating is stored with a verbatim quote from the statement that justifies it, so every score on this site can be audited by reading the quote. The label combines two things: how closely the capitals' average ratings sit to each other (Aligned within half a point, Mixed within one and a half, otherwise Divergent), and whether that shared position actually backs the goal. Three capitals in lockstep opposition to a goal are in agreement, not alignment with Weimar's aims — that case is labelled Aligned against goal, not Aligned. A shared average too close to neutral to call support or opposition is labelled Noncommittal. No black-box similarity math is involved in the headline scores.

LLM enrichment

Relevance classification, position extraction, and stance rating are all done by Gemma 4, an open-weight language model, run through Ollama — on a local machine during development and via Ollama Cloud in the automated pipeline. The model is asked which countries and issue areas each item concerns, for a one-sentence position summary per topic, a stance rating, and the quote that supports it. Its output is stored separately from the raw scraped text (under data/enriched/), and both are committed to git.

Storage and provenance

Each event is split across two YAML files committed to git, both keyed by the same relative path {source}/{YYYY-MM}/{date}-{hash}.yaml: the raw scraped fields live under data/events/ (title, text, source_url, source_published_at, ingested_at), and the computed classification plus the LLM extracted block live under data/enriched/. The {hash} is the first eight hex characters of a SHA-256 hash of the source URL and title, providing automatic deduplication. The full history is auditable via git log.

Every file also records how it was produced, so a native-language item is never mistaken for an English fallback. Each raw event carries collection (native or fallback) and collection_method (rss, html, or a Wayback backfill); each enriched sidecar carries enriched_by — the model_id, the prompt_version that produced its ratings, and whether it was enriched locally or in GitHub Actions. Values on data collected before this tracking was added are reconstructed from git history.

Corrections policy

Stance ratings are model-assigned and always quote-backed, but models make mistakes. If a rating looks wrong, if a quote is out of context, or if a statement was misclassified, please report it — corrections are reviewed and the underlying YAML is corrected in git, so the fix is itself part of the public record.

Caveats
  • Sources are ingested in their native language (German, French, Polish) — the full output of each institution, not the thinner English-translation sections where those even exist. Position summaries are extracted in English; stance evidence quotes stay in the original language.
  • If a native-language feed or listing becomes unreachable, ingestion falls back to an English section (where one exists) until it recovers, so a few events may carry English text.
  • Only the German MFA and France Diplomatie offer RSS — the other four sources are HTML scrapers that may break if site structures change.
  • Historical meetings data (pre-pipeline) is hand-curated from public records.
